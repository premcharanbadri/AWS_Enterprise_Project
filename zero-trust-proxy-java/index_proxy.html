System 2: Zero-Trust Data Mesh Proxy

1. What this actually does

Think of this system as a security checkpoint sitting between our company's employees and our production databases. When someone runs a SQL query, it hits this proxy first. The proxy checks their ID badge (a JWT token).

If a System Admin makes a request, the proxy gets the data and hands it right back. But if a Data Analyst makes a request, the proxy gets the data, actively scans the JSON response, scrubs out any credit card numbers or Social Security Numbers (replacing them with XXXX), and only hands back the masked data. It modifies network traffic on the fly without the database or the downstream application knowing it happened.

2. Why I built it

Data analysts need access to production tables to do their jobs, but they absolutely should not be looking at unmasked PII (Personally Identifiable Information). Usually, companies rely on individual app developers to remember to write masking logic in their code. Developers always forget, which leads to massive compliance breaches.

I built this to take data security entirely out of the hands of app developers. By centralizing it at the network layer, a single security rule instantly applies to the entire engineering organization.

3. The system logic (Why Java & Spring?)

You might wonder why I used Java here, while I used Python for the DLQ and C++ for the Query Router. It's about using the right tool for the infrastructure layer.

Network proxies must handle thousands of concurrent connections efficiently. Python's Global Interpreter Lock (GIL) makes highly concurrent network I/O a nightmare. Java's multithreading model is built exactly for this. Furthermore, the Spring Boot ecosystem provides a class called OncePerRequestFilter, which gave me the exact hook I needed to inject cryptographic token validation strictly before any routing logic occurs. I didn't use an off-the-shelf service mesh like Envoy because I needed deep, custom JSON payload inspection, which requires custom business logic.

4. Real-world bugs I had to fix

Getting this to pass a rigorous code review required patching several critical flaws:

  • The Aggressive Regex: My initial masking engine just looked for any 13 to 16-digit sequence. It ended up permanently corrupting non-PII enterprise identifiers like long database primary keys and National Provider Identifiers (NPIs). I had to tighten the regex automata to strictly look for standardized credit card chunking patterns to stop the false positives.
  • Silent Testing Failures: Maven was silently ignoring my Data Loss Prevention (DLP) test suite because the file was in the wrong directory. Once I moved it and forced the tests to run, I realized my test assertions were fundamentally mismatched with the actual engine's output. I had to realign the test suite to reflect production reality.
  • Auth Bypasses: For local development, I had a simple string-matching bypass in the JWT filter. A reviewer rightly flagged this as a critical vulnerability if it accidentally shipped. I documented it explicitly and mapped out the production upgrade path to AWS Cognito JWKS.

5. Are these "novel" problems?

A colleague recently asked me: "These problems seem solved, are these novel projects or did you replicate them?"

They are not novel. They are fundamental enterprise patterns. Startups reward engineers for building "novel" product features; massive tech companies like AWS reward engineers for deeply understanding how to implement foundational, solved problems safely from scratch. Building a zero-trust proxy proves I understand the low-level mechanics of network security, cryptography, and payload manipulation, rather than just knowing how to buy a vendor tool that does it for me.

6. The Bottom Line

This proxy is a real win for compliance and developer velocity. It helps with SOC2 and PCI by masking PII right at the network layer, so the rule lives in one place instead of being copied into every app. Application developers can move faster because they don't have to worry about accidentally leaking credit card numbers—the infrastructure has their back.
